\begin{abstract}
LLM-as-a-judge evaluation is increasingly used to score open-ended model outputs, but multilingual evaluation often hides a consequential implementation choice: which language the judge sees, which language the rubric uses, and whether non-English text is translated into English before judging. We call this choice the \emph{judge protocol}. We study protocol sensitivity on SEAHORSE multilingual summarization labels, source-grounded XLSum examples, and a WMT MQM translation-quality contrast. Across 580 main-experiment base examples and 2,560 judge responses, a \texttt{gpt-4o-mini} judge gives materially different scores under direct English-rubric judging, target-language rubrics, explicit English-pivot judging, and bilingual rubrics. The largest shifts appear on target-language form dimensions: Vietnamese grammar scores differ by 1.20 points on a 1--5 scale between target-language-rubric and English-pivot protocols. English pivoting is not a safe default: it significantly reduces alignment on Turkish comprehensibility, and follow-up experiments with \texttt{gpt-4.1-mini}, \texttt{gpt-5.4-mini}, and \texttt{gpt-5.5} reproduce the key failure. A small evidence-preserving bilingual protocol recovers the worst \texttt{gpt-5.5} pivot failures. Semantic and reference-based MT settings show smaller or more model-dependent effects, clarifying the boundary of the claim. We argue that global evaluation papers should report judge protocol, per-language alignment, same-item protocol shifts, calibration behavior, parse rates, and token cost instead of a single aggregate judge score.
\end{abstract}

\section{Introduction}

LLM-as-a-judge evaluation has become an attractive default for open-ended generation. A benchmark designer can write a rubric, send generated outputs to a strong model, and obtain scalable scores without recruiting human annotators for every system comparison. This workflow has been used for chat evaluation, instruction following, summarization, and machine translation, and prior work shows that LLM judges can correlate with human preferences or task-specific human ratings in many settings \citep{zheng2023judging,liu2023geval,kocmi2023large}. The practical appeal is obvious: LLM judges are cheap, fast, flexible, and easy to add to evaluation pipelines.

The workflow is less straightforward when the outputs are multilingual. In an English-only benchmark, the judge usually sees the instructions, rubric, candidate response, and any reference material in the same language. In a multilingual benchmark, these surfaces can be separated. A benchmark builder can ask an English prompt to judge a non-English output directly; translate the output, source, or reference into English and judge the translation; translate the rubric into the target language; or provide bilingual instructions. These choices are often treated as prompt-engineering details. We argue that they are not details: they determine what evidence the judge can see.

The risk is clearest when the evaluation target is not purely semantic. English-pivot judging may help an English-dominant model process content, but the translation step can also remove exactly the evidence needed to judge grammar, fluency, repetition, morphology, register, or naturalness in the target language. A disfluent Vietnamese or Turkish summary can become a clean English sentence after translation; a good original can be damaged by a noisy pivot translation. Even for semantic adequacy, the rubric language and translation pipeline can change score calibration. A multilingual benchmark score is therefore not just a property of the candidate output. It is a score produced by a \emph{judge protocol}: the complete procedure that specifies the language of the rubric, the language of the evidence, and whether any text is translated before judging.

This paper asks a practical question for global benchmark builders: \emph{when a paper reports an LLM judge score for multilingual outputs, how sensitive is that score to the language protocol used to obtain it?} We do not train a new judge or claim that one prompt is universally optimal. Instead, we treat protocol choice as an experimental factor that should be measured and reported.

We study three settings with existing human labels. First, we use SEAHORSE candidate-quality labels for multilingual summaries, focusing on candidate-visible dimensions where target-language form evidence should matter. Second, we reconstruct source-grounded XLSum examples for the SEAHORSE main-ideas dimension to test a more semantic summarization setting. Third, we use a WMT MQM translation-quality contrast to ask whether machine-translation evaluation behaves differently when a reference is available. We compare direct English-rubric judging, target-language-rubric judging, explicit English-pivot judging, and bilingual-rubric judging; for high-risk cells, we also test an evidence-preserving bilingual mitigation.

Our contributions are:
\begin{itemize}
    \item We define \emph{protocol sensitivity} for multilingual LLM-as-a-judge evaluation and operationalize it with same-item score shifts, by-language human alignment, language gaps, calibration diagnostics, parse rates, and token cost.
    \item We release a benchmark scaffold covering 580 main base examples across candidate-quality, source-grounded semantic, and WMT MQM translation-quality settings, plus stronger-judge, modern-judge, and mitigation experiments.
    \item We show that English pivoting is not a safe default: it significantly hurts Turkish comprehensibility in the main run, and the same failure mode appears under stronger and current frontier judges.
    \item We propose an evidence-preserving bilingual judging protocol that recovers the worst modern-judge pivot failures on target-language form dimensions.
    \item We provide a Global Judge Reporting Card that makes multilingual judge protocols auditable rather than hidden prompt details.
\end{itemize}

Our main finding is not that one protocol always wins. The important result is that protocol choice can change both absolute scores and human alignment, and the direction depends on language, task, dimension, and judge model.

\section{Related Work and Positioning}

\paragraph{Automatic evaluation before LLM judges.}
NLG and MT evaluation have long relied on automatic metrics such as BLEU, ROUGE, and METEOR, which compare system outputs to references using token or n-gram overlap \citep{papineni2002bleu,lin2004rouge,banerjee2005meteor}. Later learned metrics such as BERTScore, BLEURT, and COMET improved correlation with human judgments by using contextual representations or human-rated training data \citep{zhang2020bertscore,sellam2020bleurt,rei2020comet}. These metrics are efficient and reproducible, but open-ended tasks often require evaluation criteria that are hard to reduce to reference overlap alone: factual consistency, instruction following, style, safety, cultural appropriateness, and target-language form.

\paragraph{LLM-as-a-judge evaluation.}
LLM judges extend automatic evaluation by using natural-language rubrics and model reasoning as the scoring interface. G-Eval uses prompted GPT-4-style evaluators for NLG quality, while MT-Bench and Chatbot Arena popularized LLM judging for multi-turn assistant responses \citep{liu2023geval,zheng2023judging}. Other work explores trained or open evaluator models such as Prometheus, meta-evaluation benchmarks such as LLMBar, and broad surveys of LLM judge design choices and failure modes \citep{kim2023prometheus,zeng2023evaluating,li2024llmsjudgesurvey}. Our paper follows this literature in treating LLM judges as useful but imperfect instruments; our focus is the multilingual language protocol used to present evidence to the instrument.

\paragraph{Judge bias and protocol sensitivity.}
A growing literature shows that LLM judges are sensitive to evaluation protocol. Reported issues include position bias, verbosity or length bias, self-preference, model-family effects, and prompt sensitivity \citep{wang2023large,shi2024judging,dubois2024length,wataoka2024self,bavaresco2024llms}. ReIFE further shows that evaluation protocols can interact with the underlying judge model and dataset \citep{liu2024reife}. We study an analogous but underreported axis: language protocol. Instead of asking whether answer order or length changes a judge decision, we ask whether the judge decision changes when the same multilingual evidence is shown directly, translated into English, rubric-translated, or shown with bilingual instructions.

\paragraph{Multilingual and global evaluation.}
Multilingual evaluation has expanded well beyond English through benchmarks such as XTREME and XLSum \citep{hu2020xtreme,hasan2021xlsum}. Recent global benchmarks emphasize that translation alone does not make an evaluation culturally or linguistically neutral. Global-MMLU studies cultural and linguistic biases in multilingual evaluation; CulturalBench and WorldValuesBench evaluate cultural knowledge and value awareness; SEA-HELM builds a holistic evaluation suite around Southeast Asian languages and cultures \citep{singh2024global,chiu2024culturalbench,zhao2024worldvaluesbench,susanto2025seahelm}. These efforts motivate our central claim: if the benchmark itself must be localized and audited, the judge protocol used to score it must also be localized and audited.

\paragraph{Human evaluation for summarization and MT.}
Summarization and translation evaluation illustrate why protocol choice matters. Human evaluation studies distinguish semantic adequacy, factual consistency, fluency, grammar, and other quality dimensions \citep{fabbri2021summieval,maynez2020faithfulness}. MT evaluation has similarly moved toward expert error annotation and MQM-style judgments, where references, source context, and target-language form can all matter \citep{freitag2021experts,freitag2022results}. SEAHORSE is especially useful for our study because it provides multilingual, multifaceted human labels for summarization quality \citep{clark2023seahorse}. Recent multilingual judge work finds that LLM judges can be inconsistent across languages and that multilingual adjudication procedures can improve reliability \citep{fu2025reliable,fu2026languages}. Our contribution is complementary: we evaluate low-cost reporting choices available to any benchmark builder, rather than training a new evaluator.

\section{Experimental Setup}

\subsection{Concepts and Research Questions}

A \textbf{judge protocol} specifies the language of the rubric, the language of the evidence shown to the judge, and whether any source, candidate, or reference text is translated before scoring. We study four questions:
\begin{itemize}
    \item \textbf{Q1: Protocol sensitivity.} How much does the judge protocol change same-item scores and human alignment?
    \item \textbf{Q2: Boundary.} Are protocol effects larger for target-language form dimensions than for semantic or reference-based MT judgments?
    \item \textbf{Q3: Robustness and mitigation.} Do stronger/current judges or an evidence-preserving bilingual prompt remove the failure?
    \item \textbf{Q4: Reporting.} What should future multilingual LLM-judge papers report so their conclusions are auditable?
\end{itemize}

\subsection{Datasets and Sampling}

\paragraph{Candidate-quality summarization.}
We sample from the SEAHORSE test split \citep{clark2023seahorse}. The main run uses English (\texttt{en-US}), Spanish (\texttt{es-ES}), Turkish (\texttt{tr}), and Vietnamese (\texttt{vi}). We evaluate two candidate-visible dimensions: comprehensibility and grammar. For each language and dimension, we sample 50 balanced examples, with 25 positive and 25 negative human labels. This yields 400 base examples and 1,600 judge responses across four protocols.

\paragraph{Source-grounded summarization.}
SEAHORSE raw TSV files do not include source documents. To recover source-grounded examples, we match SEAHORSE reference summaries to raw XLSum test summaries \citep{hasan2021xlsum}. Matching recovers 151/151 Spanish, 243/244 Turkish, and 252/252 Vietnamese reference rows. We then sample 30 balanced main-ideas examples per language, yielding 90 base examples and 360 judge responses.

\paragraph{WMT MQM translation-quality contrast.}
We sample 30 balanced examples per language pair from WMT MQM human-evaluation rows: \texttt{en-de}, \texttt{en-ru}, and \texttt{zh-en}. We use year-2022 rows and derive high/low labels from within-pair MQM-score quantiles. We run both reference-based judging (source, candidate, reference) and reference-free judging (source and candidate only). Each condition uses direct judging and explicit English-pivot judging for all pairs, plus target-language and bilingual rubrics for the two non-English target pairs, giving 600 WMT judge responses total.

\paragraph{Robustness and mitigation experiments.}
The main runs use \texttt{gpt-4o-mini}. We then audit high-sensitivity cells with \texttt{gpt-4.1-mini}, \texttt{gpt-5.4-mini}, and \texttt{gpt-5.5}. The \texttt{gpt-4.1-mini} candidate experiment uses a stratified n=25-per-cell subset. The modern-judge experiment focuses on the cells most relevant to the paper's claim: Turkish comprehensibility, Vietnamese comprehensibility, Vietnamese grammar, and \texttt{zh-en} WMT reference-free translation quality. For the three candidate-quality cells, we also test P4, an evidence-preserving bilingual protocol.

\subsection{Judge Protocols}

We compare four main prompt surfaces:
\begin{itemize}
    \item \textbf{P0 Direct English rubric:} original candidate text with English instructions and an English rubric.
    \item \textbf{P1 Target-language rubric:} original candidate text with rubric wording translated into the target language when meaningful.
    \item \textbf{P2 Explicit English pivot:} candidate, source, and/or reference material translated into English before scoring.
    \item \textbf{P3 Bilingual rubric:} original candidate text with both English and target-language rubric wording.
\end{itemize}
The mitigation protocol, \textbf{P4 evidence-preserving bilingual}, keeps the original target-language candidate visible and requires the judge to cite original-language evidence before assigning a score. All calls request strict JSON with a 1--5 score, coarse label, confidence, and a short rationale. The release package includes redacted prompt files preserving system messages, rubric wording, protocol instructions, and JSON contracts.

\subsection{Metrics}

We report both alignment and diagnostic metrics. Alignment is measured with Spearman correlation between judge score and human label, and AUROC for ranking human-positive examples above human-negative examples. Same-item protocol shift is the mean absolute score difference between two protocols on identical base examples:
\begin{equation}
    \Delta_{p,q}^{\mathrm{score}} = \frac{1}{n}\sum_{i=1}^{n} |s_{i,p} - s_{i,q}|.
\end{equation}
For paired comparisons, we report item-level AUROC deltas. We also report cross-language gaps, post-hoc threshold calibration, JSON parse rate, returned token usage, and approximate API cost. Calibration uses small in-group calibration splits to choose a score threshold, then evaluates held-out balanced accuracy.

\section{Results}

\paragraph{Reading guide.}
Rather than present every metric cell in the main text, we organize the results around four takeaways. Table~\ref{tab:headline_results} contains the most important cells; Table~\ref{tab:boundary_results} shows where the claim weakens or becomes task-dependent; Table~\ref{tab:protocol_instability_summary} summarizes instability across all main cells.

\subsection{Takeaway 1: English Pivoting Breaks Form-Sensitive Judgments}

The strongest failures occur when the human label depends on target-language form. On Turkish comprehensibility, the main \texttt{gpt-4o-mini} judge aligns well with human labels under direct judging, but explicit English pivoting drops AUROC from 0.84 to 0.69. On Vietnamese grammar, the average same-item score shift between target-language-rubric and pivot judging is 1.20 points on a 1--5 scale. These are not small calibration differences: the pivot protocol changes what evidence the judge sees.

Qualitative inspection supports this mechanism. In one Vietnamese human-negative comprehensibility item, target-language judging scores the original summary 2, while the English pivot ``Prepare materials'' is scored 5 because the translation is clear but has removed the target-language error signal. Conversely, in a Turkish human-positive comprehensibility item, direct, target-language, and bilingual protocols all score 5, but the pivot translation repeats the sentence and the pivot judge assigns 1. Translation can therefore both clean away real errors and introduce new artifacts.

\subsection{Takeaway 2: Modern Judges Reduce Some Noise but Preserve the Core Failure}

The failure is not just a \texttt{gpt-4o-mini} artifact. The \texttt{gpt-4.1-mini} experiment reproduces the same pattern: Turkish comprehensibility drops from 0.91 AUROC under direct judging to 0.59 under pivot, and Vietnamese grammar drops from 0.81 to 0.47. The modern-judge experiment is even more important for the submission story. Under \texttt{gpt-5.5}, explicit pivot is still much worse than direct judging on Turkish comprehensibility (AUROC delta -0.26) and Vietnamese grammar (-0.47). WMT \texttt{zh-en} reference-free judging also favors direct source-visible evaluation (-0.18).

\subsection{Takeaway 3: Evidence-Preserving Bilingual Judging Is a Simple Mitigation}

P4 is designed for the failure mode observed above: it keeps original-language evidence visible and asks the judge to cite that evidence before scoring. This small change substantially recovers the worst \texttt{gpt-5.5} pivot failures. On Turkish comprehensibility, P4 improves over pivot by +0.27 AUROC and reaches 1.00 AUROC on the focused subset. On Vietnamese grammar, P4 improves over pivot by +0.50 AUROC and exceeds direct and bilingual judging on the same subset. This does not prove P4 is universally optimal, but it gives a practical recommendation: for target-language form dimensions, do not ask the judge to score only an English translation; keep the original evidence visible and require evidence citation.

\subsection{Takeaway 4: The Claim Has Useful Boundaries}

Protocol sensitivity is strongest for target-language form, but the direction is not universal. Source-grounded main-ideas evaluation shows smaller and more language-dependent effects. Spanish main ideas are best under explicit pivot, while Turkish and Vietnamese are best under bilingual judging; Vietnamese remains weak under all protocols. WMT MQM gives an additional boundary. Reference-based MT judging shows smaller direct-vs-pivot differences, plausibly because the reference constrains the judgment. Reference-free MT exposes stronger protocol effects, especially for \texttt{zh-en}. This boundary is important: the conclusion is not ``never translate.'' The conclusion is ``the protocol is part of the evaluation and must be reported.''

\subsection{No Single Protocol Dominates}

Across 17 main-experiment task/language cells, the best AUROC protocol is not direct in 11 cells, explicit pivot is the worst protocol in 10 cells, and 6 cells have at least one strongly supported paired AUROC change. The point is not to crown a universal winner. The point is that a multilingual judge score is incomplete without the protocol that produced it.

\section{Analysis and Recommendations}

\paragraph{Protocol choice changes what evidence is available.}
The clearest failures occur when the human label depends on target-language form. Pivot translation can turn a target-language surface-form problem into fluent English, or introduce English artifacts unrelated to the original candidate. For such dimensions, original-text judging with direct, bilingual, or evidence-preserving bilingual rubrics is safer than treating English pivot as the default.

\paragraph{Bilingual rubrics help, but evidence preservation is the key.}
Bilingual judging often improves over explicit pivot in high-sensitivity cells, but it is not universally best. P4 suggests a more targeted principle: when the evaluation criterion depends on form in the target language, ask the judge to inspect and cite the original-language evidence before scoring. This turns a vague ``be multilingual'' prompt into an auditable judging procedure.

\paragraph{Aggregate scores hide language gaps.}
Protocol choice changes disparity across languages. In candidate-quality comprehensibility, explicit pivot has a Spearman language gap of 0.63, from Vietnamese 0.09 to English 0.71. Bilingual judging reduces the same gap to 0.30. In source-grounded main ideas, explicit pivot again has the largest language gap, 0.51, from Vietnamese 0.00 to Spanish 0.51. A single aggregate judge score would hide these failures.

\paragraph{Calibration is a diagnostic, not a repair.}
Post-hoc threshold calibration helps most in the source-grounded semantic setting, where judge scores are compressed low: with balanced human labels, the fixed score $\geq 4$ threshold marks only 8.3\% of examples as good, and the modal best in-group threshold is 2. In candidate-quality and reference-based WMT, calibration is neutral or slightly negative on average. Calibration can reveal score compression, but it does not solve the underlying protocol problem.

\paragraph{Repeatability is not the main explanation.}
The historical n=20 pilot and the n=50 main run overlap on 42 original-text prompt calls with identical prompt text. Across these repeated calls, exact score agreement is 92.9\%, mean absolute score delta is 0.07, and the maximum score delta is 1. This is much smaller than the main protocol shifts, such as 0.96 for Vietnamese comprehensibility and 1.20 for Vietnamese grammar. By contrast, repeated explicit-pivot prompt IDs with regenerated translation text show lower agreement, suggesting that translation-pivot workflows add another source of pipeline volatility.

\paragraph{Global Judge Reporting Card.}
We recommend that multilingual LLM-as-a-judge papers report: judge model and version; prompt/rubric language; whether source, candidate, or reference texts were translated; whether target-language form dimensions were judged in the original language; per-language human alignment; same-item protocol-shift metrics; language gaps for each protocol; calibration split size and held-out effect; parse failure rate; returned token usage; and approximate cost per 1,000 judgments. Token usage is the stable accounting quantity; dollar values should be refreshed against current provider pricing before camera-ready release.

\section{Limitations}

The evidence is deliberately bounded. All main judge calls and all stronger/modern experiments use OpenAI-hosted models rather than independent model families. Target-language rubric translations are pragmatic pilot translations, not native-speaker validated. The source-grounded setting covers one semantic dimension and three non-English languages. The WMT MQM contrast is small, covers one year and three language pairs, adds target/bilingual rubrics only for the two non-English target pairs, and uses quantile-derived binary labels from continuous MQM scores. The targeted WMT stronger-judge and modern-judge experiments are robustness checks rather than definitive model-family comparisons. The reference-free WMT condition reuses the same MQM-derived labels, so it is a protocol stress test rather than a new human annotation target. SEAHORSE labels are binary yes/no labels for each dimension, while LLM judges produce 1--5 scores; thresholding and calibration are therefore part of the evaluation design.

\section{Conclusion}

Multilingual LLM-as-a-judge evaluation is under-specified without the judge protocol. The same item can receive different scores depending on whether the judge sees original text, translated text, target-language instructions, bilingual instructions, or explicit original-language evidence requirements. English-pivot judging can help in some semantic and reference-based settings, but it can also erase target-language evidence, introduce translation artifacts, and increase language gaps. The practical lesson is simple: do not let a single hidden English-pivot judge score stand in for global evaluation. Report protocol sensitivity, and preserve the evidence needed for the language being judged.
